# Node.js + TypeScript Application — Setup and Implementation Plan

## 1. Initial commands (run in order)

Run these in your project folder (`jwt-auth-backend`):

```powershell
# Create package.json with defaults (no prompts)
npm init -y

# Install TypeScript and type definitions for Node.js (dev dependencies)
npm install -D typescript @types/node ts-node nodemon

# Initialize TypeScript config
npx tsc --init
```

Then adjust `tsconfig.json` (see step 2). Optionally add a simple script runner:

```powershell
npm install -D tsx
```

---

## 2. Configure TypeScript

Edit the generated [tsconfig.json](tsconfig.json) so it’s suitable for a Node app:

- `**"outDir": "dist"**` — compiled JS output folder
- `**"rootDir": "src"**` — source folder (create `src/` and put `.ts` files there)
- `**"strict": true**` — keep strict type-checking
- `**"esModuleInterop": true**` — easier CommonJS/ESM interop
- `**"skipLibCheck": true**` — faster builds (optional)

---

## 3. Project structure

Create this layout:

```
jwt-auth-backend/
├── src/
│   └── index.ts      # Entry point
├── dist/             # Generated by tsc (add to .gitignore)
├── package.json
├── tsconfig.json
└── .gitignore
```

---

## 4. package.json scripts

Add in `package.json` under `"scripts"`:

| Script      | Command                                      | Purpose                                 |
| ----------- | -------------------------------------------- | --------------------------------------- |
| `build`     | `tsc`                                        | Compile TypeScript to `dist/`           |
| `start`     | `node dist/index.js`                         | Run compiled app                        |
| `dev`       | `ts-node src/index.ts` or `tsx src/index.ts` | Run without pre-compiling (development) |
| `dev:watch` | `nodemon --exec ts-node src/index.ts`        | Auto-restart on file changes            |

Use either `ts-node` or `tsx` for `dev`; both work with the setup above.

---

## 5. Minimal entry point

In [src/index.ts](src/index.ts):

- Import what you need (e.g. `http` or `express` later).
- Start an HTTP server or log a message so you can confirm the app runs with `npm run dev` or `npm run build` then `npm start`.

---

## 6. .gitignore

Add at least:

- `node_modules/`
- `dist/`
- `.env` (if you use env vars)
- `*.log`

---

## 7. Implementation order (process plan)

1. **Run initial commands** — `npm init -y`, install deps, `npx tsc --init`.
2. **Update tsconfig.json** — set `outDir`, `rootDir`, and options above.
3. **Create `src/` and `src/index.ts`** — minimal “Hello World” or HTTP server.
4. **Add scripts to package.json** — `build`, `start`, `dev` (and optionally `dev:watch`).
5. **Add .gitignore** — `node_modules/`, `dist/`, `.env`, logs.
6. **Verify** — `npm run dev` (or `npm run build` then `npm start`) and confirm it runs.

After this, you can add Express/Fastify, JWT auth, routes, and database logic on top of this base.

---

## Optional: Express for API backend

If you want an HTTP API (fits “jwt-auth-backend”):

```powershell
npm install express
npm install -D @types/express
```

Then in `src/index.ts`: create an Express app, define a simple GET route (e.g. health or `/`), and call `app.listen(port)`. You can add JWT and routes in later steps.

## Summary

| Step | Action                                                                                            |
| ---- | ------------------------------------------------------------------------------------------------- |
| 1    | `npm init -y` then install TypeScript, `@types/node`, `ts-node`, `nodemon` (and optionally `tsx`) |
| 2    | `npx tsc --init` and edit `tsconfig.json` (outDir, rootDir, strict, etc.)                         |
| 3    | Create `src/index.ts` and minimal server or console script                                        |
| 4    | Add `build`, `start`, `dev` (and optionally `dev:watch`) in package.json                          |
| 5    | Add .gitignore and run `npm run dev` to verify                                                    |

This gives you a clean, runnable Node.js + TypeScript app and a clear path to add JWT auth and API logic next.
